構文・機密ファイル混入のチェックを通過したものだけをステージし、意味のあるコミットメッセージで自動コミットする。

## 対象
- 引数 `$ARGUMENTS` にコミットメッセージを指定できる。未指定の場合は差分から自動生成する。

## 手順

### Phase 1: 品質チェック

1. **Python 構文チェック**（必須）:
   ```bash
   python3 -m py_compile app.py main.py slide_image.py
   ```
   エラーがあれば即座に報告してコミットを中止する。

2. **ruff**（インストール済みなら）:
   ```bash
   command -v ruff && ruff check .
   ```
   違反があれば報告してコミットを中止する。

3. **mypy**（インストール済みなら）:
   ```bash
   command -v mypy && mypy app.py main.py slide_image.py
   ```
   致命的な型エラーがあればコミットを中止する（警告のみなら継続）。

4. **pytest**（`tests/` ディレクトリがあれば）:
   ```bash
   [ -d tests ] && pytest -q
   ```
   テストが落ちていればコミットを中止する。

### Phase 2: 差分確認

1. `git status` で変更ファイルを確認する。
2. `git diff` で変更内容を確認する。
3. **機密ファイル混入チェック**: 以下が含まれていないことを確認する。含まれていれば**強制的に除外し**マスターに警告する:
   - `.env` / `.env.*`
   - `credentials.json`
   - `*.key` / `*.pem`
   - `**/*secret*` / `**/*token*`
4. **検証用残骸チェック**: ルート直下に `*.png` / `*.jpeg` の検証スクショが残っていないか確認する（`static/` 内の本番アセットは除外）。残骸があればマスターに削除を促す。
5. **`photos/` チェック**: 検証用にアップロードした画像が `photos/` に残っていないか確認する（`.gitkeep` 以外）。

### Phase 3: コミットメッセージ生成

- `$ARGUMENTS` が指定されている場合: そのメッセージをそのまま使用する。
- 未指定の場合: 差分を分析し、以下のフォーマットでメッセージを生成する。

```
<種別>: <変更の理由または内容（英語または日本語・50文字以内）>

種別: feat / fix / refactor / test / chore / ci / docs
```

既存リポジトリの履歴を `git log --oneline -10` で参照し、日本語/英語のトーンを揃える。

### Phase 4: ステージ & コミット

1. `git add` で変更ファイルをステージする（機密ファイル・検証用残骸を除く）。
2. 以下でコミットする:

```bash
git commit -m "$(cat <<'EOF'
<生成したメッセージ>

Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>
EOF
)"
```

3. コミット成功後、コミットハッシュとメッセージを報告する。

## 注意

- 構文チェックが失敗した場合、コミットは実行しない。エラー内容を報告し、修正を促す。
- コミットは push しない。push は明示的に指示された場合のみ行う。
- 1コミット = 1つの意味のある変更単位を守る。無関係な変更が混在している場合は分割を提案する。
- `--no-verify` は使わない。
- `--amend` は push 済みのコミットには絶対に使わない。
